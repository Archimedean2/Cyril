# Repo Recon Checklist

Before requesting implementation guidance from the senior dev, inspect the repo and capture:

## Stage / planning
- Current stage from `PROGRESS.md`
- Relevant stage requirements from `STAGES.md`
- Relevant feature scope from `FEATURES.md`
- Relevant constraints from `SCOPE.md`

## Design / UX
- Relevant guidance from:
  - `WIREFRAMES.md`
  - `DESIGN_SYSTEM.md`
  - `UI_TOKENS.md`
  - `COMPONENT_MAP.md`
  - `docs/interactions/*.md`

## Implementation
- entry points
- layout shell files
- components directly involved
- current state management
- persistence path if relevant
- test ids already present
- existing tests covering this area

## Risk scan
- likely regressions
- schema implications
- editor-framework constraints
- whether the task touches future-stage surfaces indirectly

## Deliverable
Produce a structured Junior-to-Senior Handoff using the required template.
